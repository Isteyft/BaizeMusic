# React 音乐播放器需求文档

## 1. 文档目标
在当前 monorepo（`pnpm workspace + turbo`）架构下，定义一个可落地的 React 音乐播放器需求，作为后续设计与开发依据。

## 2. 项目范围
- 前端应用：`apps/frontend/website`（React 播放器 UI 与交互）
- 后端应用：`apps/backend/server`（扫描本地 `./music` 目录并提供媒体元数据接口）
- 共享类型：`packages/types`（播放器相关 TypeScript 类型）
- 可选工具库：`packages/utils`（时间格式化、歌词解析等）

## 3. 目标用户与使用场景
- 用户：本地运行项目的开发者/个人用户。
- 场景：启动服务后，播放器自动识别 `./music` 中音频文件，用户可浏览、播放、调节音量、拖动进度，并查看歌词与封面。

## 4. 功能需求

### 4.1 音乐文件扫描
- 系统应在启动时扫描项目根目录下 `./music` 文件夹。
- 支持音频格式：`mp3`、`flac`、`wav`、`m4a`、`ogg`（首版至少实现 `mp3`、`flac`、`wav`）。
- 返回字段至少包含：
  - `id`
  - `title`
  - `artist`
  - `album`
  - `duration`
  - `filePath`（或可访问 URL）
  - `cover`（封面 URL 或内嵌封面提取结果）
  - `lyric`（歌词来源路径或歌词文本）

### 4.2 播放控制
- 支持播放/暂停。
- 支持上一首/下一首（按当前列表顺序）。
- 支持点击列表项切换播放。
- 切换歌曲时自动重置进度并加载新歌曲元数据。

### 4.3 进度控制
- 展示当前播放时间与总时长（`mm:ss`）。
- 支持拖拽进度条并跳转到对应时间点。
- 拖拽释放后应立即生效，音频从目标时间继续播放。

### 4.4 音量控制
- 支持 0~100% 音量调节（UI 层可映射为 0~1）。
- 支持静音/取消静音。
- 音量值在页面刷新后可恢复（本地存储）。

### 4.5 歌词显示
- 支持 `.lrc` 歌词文件解析与展示。
- 默认按“歌曲同名”规则匹配（如 `song.mp3` 对应 `song.lrc`）。
- 支持逐行高亮（根据时间标签同步）。
- 无歌词时显示降级文案（如“暂无歌词”）。

### 4.6 音乐封面显示
- 优先使用音频文件内嵌封面。
- 若无内嵌封面，尝试同目录同名图片（`jpg/png/webp`）。
- 若都不存在，显示默认占位图。

## 5. 非功能需求
- 性能：
  - 200 首以内歌曲列表首次加载时间 < 2 秒（本地开发环境目标）。
  - 进度与歌词同步误差控制在 ±300ms 内。
- 可维护性：
  - 前后端接口使用统一类型定义（`packages/types`）。
  - 核心逻辑（歌词解析、时间格式化）抽取为可复用模块。
- 兼容性：
  - 首版目标浏览器：Chrome 最新稳定版。

## 6. 信息架构与页面草案
- 主页面（单页即可）：
  - 左侧：播放列表
  - 中间：封面 + 歌词
  - 底部：播放控制区（播放按钮、进度条、时间、音量）

## 7. 接口需求（建议）
- `GET /api/tracks`
  - 返回歌曲列表及元数据。
- `GET /api/tracks/:id/stream`
  - 音频流输出。
- `GET /api/tracks/:id/lyric`
  - 返回歌词文本（可选，若前端直读也可省略）。
- `GET /api/tracks/:id/cover`
  - 返回封面资源（可选）。

## 8. 异常与边界
- `./music` 目录不存在：返回空列表并给出提示。
- 音频损坏或无法解码：在列表标记不可播放并记录日志。
- 歌词时间标签异常：忽略错误行，保留可解析内容。
- 拖拽到超出范围的时间：自动裁剪到 `[0, duration]`。

## 9. 验收标准
- 能正确扫描并展示 `./music` 中支持格式的文件。
- 可完成播放、暂停、切歌、进度拖拽、音量调节。
- 封面可按优先级正确显示。
- 歌词可显示并随播放时间高亮滚动。
- 关键功能具备基础测试（至少包含歌词解析与时间格式化单测）。

## 10. 里程碑建议
- M1：完成后端扫描与歌曲列表 API。
- M2：完成前端播放器基础 UI 与播放控制。
- M3：完成歌词同步、封面策略、音量与进度持久化。
- M4：联调、测试与文档完善。
